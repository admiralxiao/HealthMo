# 风险信号清单（Signals）与采集

更新时间：2026-01-30

---

## 1. 账号维度（Account）
强信号：
- 单日注册次数（同 device / 同 IP）
- 失败登录次数（暴力尝试）
- 邮箱未验证但高频调用高成本功能（vision）
- 同账号多设备频繁切换（短时间内）

弱信号（仅参考）：
- 时区频繁切换
- 语言/地区频繁切换

---

## 2. 设备维度（Device）
强信号：
- Jailbreak/注入/调试态（risk signal，不直接封）
- 模拟器环境（若你不支持，可直接限制）
- App Attest 失败（Level1）

弱信号：
- 设备型号/OS 过旧（用于异常分群）

---

## 3. 订阅维度（Subscription）
强信号：
- receipt 显示过期/退款，但客户端仍尝试访问 premium API
- 频繁 restore / purchase fail
- 多账号在同 device 上连续 trial（典型滥用）

---

## 4. 成本维度（Vision/LLM）
强信号：
- 单用户/单 device 在短时间内大量 photo requests
- photo requests 失败率异常高（可能在 fuzz 图片）
- 同 IP 多账号共享刷接口

---

## 5. API 维度（Traffic）
强信号：
- 高频请求（超过阈值）
- 重放（nonce/timestamp 不合理）
- UA/headers 异常

---

## 6. MVP 需要埋点/日志字段（最小）
- user_id, device_id, session_id
- ip_hash（不可逆 hash）
- locale, timezone
- request_id, latency_ms, status_code
- vision_request_count/day
- subscription events：subscribe_* / restore_*
- risk_flags（客户端上报：debugger/jailbreak signal 等，仅 signal）

> 注意：不要收集“过度个人信息”。IP 建议 hash 存储，避免隐私风险。
